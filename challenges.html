Challenge 2: Enhancing the Camera operation

Challege 2.1: Front camera property issues

After a quick research,I found that in the class Camera.java, the useFront property use the intent ("android.intent.extras.CAMERA_FACING", 1) 

// NOTE: This uses an undocumented, testing feature (CAMERA_FACING).
                // It may not work in the future.
  if (useFront) {
    intent.putExtra("android.intent.extras.CAMERA_FACING", 1);
  }

It seems that this feature is not officially documented which makes it a random feature depending on the phones. For example in my One Plus One running Android Cyanogen i can test the useFront() and it's working perfectly. But on another Android phone it won't. Furthermore, there is a method added in the latest sources from Android Open Source Project more precisely in the VideoCamera class named getCameraFacingIntentExtras() which seems to be an official update of the previous functionality. Therefore that can explain why the userFront property is not working anymore.

Challege 2.2: Automatic camera design

I think sticking to the Barcode component design.That is to create an Activity class like the AppInvCaptureActivity.java class that will be linked to our component class. The difference will be that in the handler of the Activity class instead of decoding the picture taken in order to return the Barcode Text we'll just have to return the image taken.

The Activity class will create, open,launch the camera and take the picture using the takePicture(ShutterCallback, PictureCallback, PictureCallback, PictureCallback) method. The documentation can be found here. Also, we can use a timer to trigger the call of the method on the camera. Furthermore, we can use the ShutterCallback parameter (Callback interface used to signal the moment of actual image capture.) to play a sound so that the user will know that the picture was taken.

Finally in our component class, we'll just have to create an explicit Intent and pass it the Activity class we've talked about. And we will take in the void resultReturned(int requestCode, int resultCode, Intent data) method, the picture taken and eventually raised an event public void AfterPicture(String image) that will indicate that a photo was taken with the camera and provides the path to the stored picture.

We can add 2 properties: @SimpleProperty(The time the user will wait for the picture to be taken once the function is called) and @SimpleProperty(The number of pictures that will be taken (default = 1) )

Why not adding a TakePicture( int time, int numberOfPictures) method also .
